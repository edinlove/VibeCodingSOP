### 参考项目（External Reference Projects）
要主动分析的项目的那些部分

todo-update::{{具体项目的那些部分，比如架构、接口、数据库}}
1. 学习架构设计
2. 学习业务流程
3. 学习代码组织方式
4. 学习算法实现思路
5. 学习关键代码片段

todo-update::{{分析之后的输出结果}}
输出：  
1. 这些项目解决了什么问题  
2. 哪个项目与当前需求最接近  
3. 哪些代码值得参考  
4. 哪些架构值得参考  
5. 最终准备采用哪些实现方案

### 禁止行为：

- 不允许直接 import 参考目录下任何项目
- 不允许依赖参考目录下任何项目
- 不允许修改参考目录下任何项目
- 不允许在生产代码中引用参考目录下路径
- 禁止直接复制超过50行连续代码。可以参考实现逻辑，但是不能直接搬运源代码。允许改写。

### 要求：

- 优先理解设计思想，独立、重新实现功能
- 允许借鉴设计思想
- 允许参考算法逻辑
- 允许重写类似实现
- 必须适配当前项目架构

### 最终目标：

当参考目录被全部删除后，当前项目不受影响，仍可独立运行。
