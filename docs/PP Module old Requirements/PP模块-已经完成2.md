## 点击工具栏“Prompt”弹出的第一个Tab选项卡的功能说明

“Prompt”页面的第一个选项卡页面叫“Optimization”选项卡页面。为了减少与index.html的复杂度与耦合度，建议新创建prompt-optimization.html页面，以便后续维护与修改的时候更清晰，更简洁。

只有选中思维导图页面的节点(一个或者多个节点)，才能打开“Optimization”选项卡页面，否则Toast弹窗提示用户“必须选择节点才能实施优化。”

“Optimization”选项卡页面，分为3个区域：
分别是：
1、“元数据整合区”，位于“Optimization”选项卡页面顶部。宽度占整个页面的100%。该区域顶部有标题栏，名称为“元数据整合”，文本左对齐。
2、“工作目录配置区”，位于“Optimization”选项卡页面的左下区域。宽度占整个页面的50%。该区域顶部有标题栏，名称为“工作目录（可读写）配置”，文本左对齐。
3、“模板配置区”，位于“Optimization”选项卡页面的右下区域。宽度占整个页面的50%。该区域顶部有标题栏，名称为“模板配置”，文本左对齐。
4、“底部按钮区”，位于整个页面的底部固定的一行。该区域顶部底部有一行按钮区，包含“取消”、“Bulider-md”、“Bulider-json”按钮，所有按钮右对齐。该区域还有关键信息区域，显示了“选中节点数：xxx”、“相关节点数：xxx”、“选中文件数：xxx”、“选中文件夹数：xxx”、“选中模板数：xxx”，所有信息左对齐。中间不要用“｜”分隔。
“Optimization”选项卡页面所有区域的内容和组件，样式紧凑。其中“元数据整合区”、“工作目录配置区”、“模板配置区”随着内容的增加可能很长，如果超出页面，右侧会有滚动条。但是“底部按钮区”始终固定“Optimization”选项卡页面底部，不随着右侧滚动条向上/向下滑动。

### 元数据整合区

“元数据整合区”页面布局：
“元数据整合区”标题栏，显示“元数据整合”标题。在“元数据整合区”中配置与输出的所有内容，组成了“元数据整合内容”。标题栏上有“导出”按钮，使用“./assets/icon-table-export.png”(基于根目录)作为图标，右对齐，鼠标悬停在该图标，向下Toast显示“导出元数据整合内容”。点击该按钮，将会下载“元数据整合内容”到本地电脑。文件命名规则为“元数据整合内容 + 时间戳（年月日时分秒）.md”。
“元数据整合区”的所有操作，都是为了能够按照规则，正确地输出“元数据整合内容”的完整内容。

用户选中了思维导图页面的节点(一个或者多个节点)，针对被选中的节点这些节点，设置了两个全局规则：
规则一：“节点元数据信息的提示词内容”。该名词所代表的含义是：“节点元数据信息”中“```
与```”注释之外的所有信息，也即去掉了所有带有“：：”标记的frontmatter的信息、注释信息，而且去掉了所有空行的内容。节点的这类内容，才是“节点元数据信息的提示词内容”的全部内容。

规则二：“节点元数据整合算法”。“元数据整合内容”将会加载被选中节点的祖宗节点与子孙节点的元数据。“节点元数据整合算法”定义了一套加载“节点元数据”的顺序的规则。该规则为：① 祖宗节点：从根节点开始，到被选中节点的父节点为止，所有的节点。② 子孙节点：被选中节点的所有子节点、孙节点，一直到叶节点的所有节点。③弟节点：同层级节点中层级比被选中节点高的节点。④ 针对不同层级的节点：优先处理被选中节点的更低层级的节点（也即距离根节点最近的节点。注意，不同层级排序越靠近根节点的节点，未必是节点ID更小的节点，而是以不同层级的排序（排序，指的是节点的“Level”与“LevelOrder”）为判断依据），默认把被选中节点的所有祖父节点全部加载到排序表，也即，即使没有选中该节点的父节点、祖父节点、根节点，但是排序表中自动加载所有祖宗节点。有些节点虽然层级低，但是如果该节点的父节点、祖父节点、甚至任意祖宗节点的层级比其他节点距离根节点更近，那么优先处理该选中节点，而非不同层级的其他节点。 ⑤ 针对同层级的节点：优先处理同层级顺序最低的节点（也即同层级排序越靠近上的节点。注意，同层级排序越靠近上的节点，未必是节点ID更小的节点，而是以同层级的排序（排序，指的是节点的“Level”与“LevelOrder”）为判断依据），默认把被选中节点的所有子孙节点全部加载排序表，也即及时没有选中该节点的子节点、孙节点、以及叶节点，但是排序表中自动加载所有子孙结点。⑥ 子孙节点排序规则：优先处理层级更低的子节点，以及子节点到叶节点链路的节点。也即子节点的子孙节点、叶节点的优先级要比被选中节点的“弟节点”高。⑦ 排序顺序的优先顺序：子孙结点的优先级比“弟节点”高。

关于排序规则的举例说明：
现有思维导图节点为：
根节点“节点1”有两个子节点：“节点2”和“节点3”；
“节点2”有两个子节点：“节点4”和“节点5”；
“节点4”有两个子节点：“节点6”和“节点7”；
“节点6”有一个子节点：“节点8”；
“节点7”有一个子节点：“节点9”；
现在选中了“节点4”和“节点3”。

根据上述信息处理该思维导图中被选中节点的“元数据整合内容”的内容构成：

基于规则二的①，“节点4”的祖宗节点是：“节点1”和“节点2”；“节点3”的祖宗节点是：“节点1”。
基于规则二的②，“节点4”的子孙节点是：第一分支子孙节点“节点6”、“节点8”，第二分支子孙结点“节点7”和“节点9”。

基于规则二“节点元数据整合算法”，处理节点的排序规则，解释说明如下。
基于规则二④ ，虽然没有选中“节点2”，但是因为选中了“节点4”，所以默认加载“节点2”。优先级排序方面，“节点4”要排在“节点3”的前面（即使“节点3”的层级比“节点4”距离根节点近，但是因为“节点4”的父节点是“节点2”，所以“节点4”要排在“节点3”的前面）。
被选中节点的祖父节点的排序顺序是：“节点1”>“节点2”>“节点4”>“节点3”。
基于规则二⑤和⑥，虽然没有选中该节点的叶节点，但是默认把被选中节点的所有子孙节点全部加载排序表。被选中节点的子孙结点的排序顺序是：“节点6”>“节点8”>“节点7”>“节点9”。

“元数据整合内容”将会加载被选中节点的祖宗节点与子孙节点的元数据。基于规则二⑦，要加载选中节点的“祖宗节点”与“子孙节点”的“节点元数据信息的提示词内容”顺序为：“节点1”>“节点2”>“节点4”>“节点6”>“节点8”>“节点7”>“节点9”>“节点3”。

假设选中了多个节点，选中节点数为n，则选中节点数组为：selectNodeArray[]。按照“节点元数据整合算法”规则，先处理选中节点中排序靠前的第一个节点，也即selectNodeArray[0]的节点的“元数据整合内容”。
1、“元数据整合区”文本区，位于“元数据整合区”标题栏的下方。文本框中显示被选中节点selectNodeArray[0]的“节点元数据信息的提示词内容”。
2、“祖宗节点元数据集”配置区，位于“元数据整合区”文本区的下方。显示被选中节点selectNodeArray[0]祖宗节点的ID，每个祖宗节点前都有一个勾选框，鼠标勾选该节点框之后，也即要把被勾选节点的“节点元数据信息的提示词内容”插入到“元数据整合内容”中。勾选是可以多选的。默认只勾选父节点。
3、“子孙节点元数据集”配置区，位于“祖宗节点元数据集”配置区的下方。按照分支开展分组（从第一个子节点开始到叶节点，算第一个分支；第二个子节点开始到叶节点，算第二个分支；直到所有子节点全部遍历完成，形成了所有分支），显示被选中被选中节点selectNodeArray[0]所有子孙节点ID。每个祖宗节点前都有一个勾选框，鼠标勾选该节点框之后，也即要把被勾选节点的“节点元数据信息的提示词内容”插入到“元数据整合内容”中。勾选是可以多选的。默认勾选所有分支。

按照“节点元数据整合算法”规则，处理选中节点中排序靠前的第二个节点，也即selectNodeArray[1]的节点的“元数据整合内容”。参照selectNodeArray[0]的规则，在selectNodeArray[1]的“元数据整合区”文本区、“祖宗节点元数据集”、“子孙节点元数据集”配置区，显示对应内容。鼠标勾选该节点框之后，要把被勾选节点的“节点元数据信息的提示词内容”插入到“元数据整合内容”中。

按照这个规则类似处理所有选中节点，直到selectNodeArray[n-1]的节点也都参照selectNodeArray[0]的规则，在selectNodeArray[n-1]的“元数据整合区”文本区、“祖宗节点元数据集”、“子孙节点元数据集”配置区，显示对应内容。鼠标勾选该节点框之后，要把被勾选节点的“节点元数据信息的提示词内容”插入到“元数据整合内容”中。

上述所有内容都追加在“元数据整合内容”中的“# Core Task Instructions\n”之后。

至此，已经完整解释了如何按照规则，正确地排序、形成“元数据整合内容”的完整内容。“元数据整合区”标题栏的“导出”按钮，可以将“元数据整合内容 + 时间戳（年月日时分秒）.md”文件，下载到本地电脑。

### 工作目录配置区

“工作目录配置区”页面布局：
“工作目录配置区”标题栏，显示“工作目录（可读写）配置”标题。在“工作目录配置区”中配置与输出的所有内容，组成了“工作目录整合内容”。标题栏上有“配置”按钮，使用“./assets/icon_setting-config.png”(基于根目录)作为图标，右对齐，鼠标悬停在该图标，向下Toast显示“配置工作目录整合内容”。在开发环境，点击该按钮，将会打开基于根目录的“./prompt/promptTemplateConfig.json”的文件夹；在生产环境，点击该按钮，将会打开根目录。其中，promptTemplateConfig.json文件的“excluded_patterns”，定义了要排除的文件夹、文件、隐藏文件。
标题栏上有“导出”按钮，位置在“配置”按钮的左边，使用“./assets/icon-table-export.png”(基于根目录)作为图标，右对齐，鼠标悬停在该图标，向下Toast显示“导出工作目录整合内容”。点击该按钮，将会下载“工作目录整合内容”到本地电脑。文件命名规则为“工作目录整合内容 + 时间戳（年月日时分秒）.md”。
“工作目录配置区”的所有操作，都是为了能够按照规则，正确地输出“工作目录整合内容”的完整内容。

“工作目录配置区”标题栏的下方，是“工作目录选择区”。该区域将把把根目录下的所有文件和文件夹加载出来（注意：排除了promptTemplateConfig.json文件的“excluded_patterns”中定义的文件夹、文件、隐藏文件），每个文件夹和文件前面都有打勾选项。文件前面打勾，意味着勾选该文件；文件夹前面打勾，意味着勾选了该文件夹内的所有文件（注意：排除了promptTemplateConfig.json文件的“excluded_patterns”中定义的文件夹、文件、隐藏文件）。默认所有文件夹都折叠且不勾选，所有文件都不勾选。

“工作目录选择区”的下方，是“快捷选项配置区”。包含快捷选择项目：“全选”、“全不选”、“AI智选”。左对齐。
对各快捷选项解释如下：
“全选”，意味着工作目录空间的所有文件和文件夹前面，都打勾，都被选中。
“全不选”，意味着工作目录空间的所有文件和文件夹前面，都打不勾（即使已经被打勾的文件或文件夹，也都立即取消打勾），都不被选中。
“AI智选”，勾选该选项时，会读取“./prompt/promptTemplateConfig.json”（基于根目录）的文件中，关于“AIAutomaticallySelectsFiles”关键字的信息，该字段中的文件路径，是基于根目录的，这些文件是项目的注释与说明文档，便于AI快速分析项目结构。勾选该选项时，意味着只有“AIAutomaticallySelectsFiles”定义的文件，才会被打勾，被选中，除了这些文件的其他的工作目录空间的所有文件和文件夹，都打不勾（即使已经被打勾的文件或文件夹，也都立即取消打勾），都不被选中。

上述所有打勾的文件与文件夹，都追加在“工作目录整合内容”中的“# Workspace & File Access Rules\n## Read & Write Files Included\n”之后，意味着这些文件都是“可读写的工作空间与文件”。如果勾选的是“AI智选”，意味着除了勾选的文件路径外，还会额外追加这段文本：“- Workspace Access Decision Rule:\n I will NOT manually specify which directories or files can be read or written.\n You MUST FIRST read and analyze the "code comments and project documentation contained in the files under the above path" I provided, then AUTOMATICALLY DECIDE which files and folders are safe to read, write, modify, or create.\n - Your decision must follow these principles:\n Only read/write/modify business code, core logic, components, pages, and functional files.\n You can only perform operations within the scope you judge to be safe, necessary, and reasonable.\n Do NOT ask me for confirmation; make independent judgments based on project structure and the explanation document.”。

至此，已经完整解释了如何按照规则，正确地生成“工作目录整合内容”的完整内容。

### 模板配置区

“模板配置区”页面布局：
“模板配置区”标题栏，显示“模板配置”标题。在“模板配置区”中配置与输出的所有内容，组成了“模板配置整合内容”。标题栏上有“导出”按钮，使用“./assets/icon_setting-config.png”(基于根目录)作为图标，右对齐，鼠标悬停在该图标，向下Toast显示“模板配置整合内容”。点击该按钮，将会下载“模板配置整合内容”到本地电脑。文件命名规则为“prompt + 时间戳（年月日时分秒）.md”。
“模板配置区”的所有操作，都是为了能够按照规则，正确地输出“模板配置整合内容”的完整内容。

“模板配置区”标题栏的下方，是“模板配置选择区”，该区域是一个列表框。列表的表头分别为：“模板名称”（promptTemplateTitle）、“模板类型”（promptTemplateTypeName）、“模板标签”(promptTemplateTagName)、“模板作用域”(promptTemplateScope)、“模板ID”(promptTemplateID)、“模板版本号”(promptTemplateVersion)。列表框的每一列最后都有一个“滑动”按钮，使用“./assets/icon-draghandle.png”(基于根目录)作为图标，右对齐，鼠标悬停在该图标，向下Toast显示“拖动手柄”。上下拖动该按钮，将会调整该模板在“模板配置选择区”列表框中的排序。

“模板配置选择区”的下方，是“常用模板配置区”，该区域内所有选项都有复选框，可以多选。每一个选项，都是在“./prompt/promptTemplateLibrary.json”（基于根目录）中所有标记“ "promptTemplateFavorite": true,”的模板的名称。如果模板太多导致整行显示不下，则左右滑动显示，不分行显示。

“常用模板选择区”的下方，是“通用模板配置区”，该区域有四个下拉菜单按钮（左对齐）和一个“添加”按钮（右对齐）。
四个下拉菜单按钮分别是：
第一个下拉菜单是：“模板类型”下拉菜单。默认不选。只能单选。
第二个下拉菜单是：“模板名称-Type”下拉菜单。默认不选。只能单选。
当在“模板类型”下拉菜单中选择了模板库中的模板类型时（也即promptTemplateType关键字的promptTemplateTypeName的值），则在“模板名称-Type”下拉菜单中，可以选择到符合“模板类型”下拉菜单的“模板名称”（也即，该模板名称，在该模板类型的分组下）。

第三个下拉菜单是：“模板标签”下拉菜单。默认不选。只能单选。
第四个下拉菜单是：“模板名称-Tags”下拉菜单。默认不选。只能单选。
当在“模板标签”下拉菜单中选择了模板库中的模板标签时（也即promptTemplateTags关键字的promptTemplateTagName的值），则在“模板名称-Tags”下拉菜单中，可以选择到符合“模板标签”下拉菜单的“模板名称”。

“添加”按钮，使用“./assets/icon-list-add.png”（基于根目录）作为图标，鼠标悬停在该图标，向下Toast显示“添加模板到配置列表”。
每次点击该按钮，可以把“模板名称-Type”下拉菜单中被选中的“模板名称”对应的模板，以及“模板名称-Tags”下拉菜单中被选中的“模板名称”对应的模板，都加载到“模板配置选择区”的列表框中。

上述“模板配置选择区”的列表框中的模板的文件内容（注意不是模板的文件路径，而是模板的全文内容），按照顺序，逐一追加在“模板配置整合内容”中的“# Common Implementation Specifications\n”之后。

至此，已经完整解释了如何按照规则，正确地生成“模板配置整合内容”的完整内容。

### 底部按钮区

### 具体事项

**A. 数据结构**
```javascript
// 从父页面接收的数据
selectedNodes = [
  {
    id: 'n4',
    uuid: 'xxx',
    text: '节点4',

// 计算后的排序信息
nodeOrderMap = {

  'n2': {preorderIndex: 1, level: 1, levelOrder: 0},
  'n4': {preorderIndex: 2, level: 2, levelOrder: 0},

**B. 算法实现**
1. `computePreorderIndex(tree)` — DFS 先序遍历，给每个节点分配 `preorderIndex`

   - 按 `` ``` `` 分割文本
   - 去掉 frontmatter 块（第 1 个 `` ``` `` 到第 2 个 `` ``` `` 之间）
   - 去掉空行
   - 返回剩余内容
5. `generateIntegrationContent(selectedNodes, checkedMap)` — 生成元数据整合内容：
   - 遍历所有相关节点，按 `preorderIndex` 排序
   - 只保留 `checkedMap` 中被勾选的节点
   - 每个节点内容前添加 `# 节点标题\n`
   - 拼接为完整文本，前缀为 `# Core Task Instructions\n`

**C. UI 实现**

     - 每个节点前有复选框，默认只勾选最后一个（直接父节点）

     - 每个节点前有复选框，默认全选
3. 底部预览区：实时显示当前的"元数据整合内容"（随着勾选变化实时更新）

**D. 导出功能**
