# VibeCodingSOP — Markdown Mind Map Editor

[中文 Chinese](https://github.com/edinlove/VibeCodingSOP/blob/main/README-CN.md) | 

A mind map editor tailored for the Vibe Coding workflow, supporting Markdown import/export, node metadata management, and a Prompt Template Library. It also offers three runtime modes: standalone server, VS Code extension, and Obsidian plugin.

## Core Features

### Mind Map Core
- **Bidirectional Markdown Conversion**: Import Markdown to generate mind maps, and export mind maps to Markdown format
- **Node Metadata**: Each node supports metadata fields including Status, Priority, Mainline, Module, and AITaskJournal
- **Drag-and-Drop & Layout**: Node dragging, auto-alignment layout, zoom, and pan functionality
- **Bulk Operations**: Multi-node selection via box select, bulk editing capabilities
- **Undo/Redo**: Complete history stack for all actions

### Prompt Template Library
- **Template Management**: Import, edit, and disable Prompt templates
- **Grouped Display**: Auto-grouped by PromptType, with support for favorites and pinning
- **Version Control**: Template version number management with automatic historical version backups
- **Builder Functionality**: One-click generation of Markdown/JSON from selected node information, automatically appended to node metadata
- **History Log**: Records all files generated by the Builder, with support for appending to node content

### Multi-Platform Support
- **Standalone Server**: Launch directly with `node server.js` and access via browser
- **VS Code Extension**: Integrated into the Activity Bar, embedded via Webview iframe
- **Obsidian Plugin**: Integrated via Ribbon Icon, embedded in WorkspaceLeaf via iframe, supporting desktop (Mac/Windows)

### Additional Features
- **Table View**: Dedicated page displaying nodes in tabular format (Tabulator.js)
- **Auto-Save**: Scheduled automatic saving to files
- **Local Backups**: Automatic backups to `todo/.bak/` before saving
- **Theme Switching**: Enhanced theme support (controlled via settings toggle)

## Installation & Usage

### Option 1: Obsidian Plugin (Desktop)

1. Clone this repository locally
2. Build the plugin:
   ```bash
   npm run build:obsidian
   ```
3. Copy the build output to your Vault's plugin directory:
   ```bash
   cp -r release/vibecodingsop /path/to/Vault/.obsidian/plugins/
   ```
4. Restart Obsidian
5. Navigate to `Settings → Community plugins → Installed plugins` and enable **VibeCodingSOP**
6. Click the VibeCodingSOP icon in the left Ribbon to open

**Data Path Configuration**: Go to `Settings → Community plugins → VibeCodingSOP` to modify the data directory (default: `EdPau3/`, relative to the Vault root).

### Option 2: VS Code Extension

1. Clone this repository locally
2. Build the extension:
   ```bash
   npm run build:vscode
   ```
3. Install the generated `.vsix` file in VS Code:
   - Open VS Code
   - Click the Extensions icon at the bottom of the Activity Bar
   - Click the `...` in the top-right → `Install from VSIX...`
   - Select the generated `release/vibecodingsop-{version}.vsix`
4. After installation, the VibeCodingSOP icon will appear in the Activity Bar; click to launch

### Option 3: Standalone Run

```bash
pnpm install
pnpm start
```

Then open your browser and navigate to `http://localhost:5000`

## Unified Build System

This project provides a one-click build script to generate both VS Code and Obsidian plugins simultaneously:

```bash
# One-click build for both plugins
npm run build:release

# Build only the VS Code extension
npm run build:vscode

# Build only the Obsidian plugin
npm run build:obsidian
```

Build outputs are located in the `release/` directory:

```
release/
├── vibecodingsop-{version}.vsix   ← VS Code extension
└── vibecodingsop/                  ← Obsidian plugin directory (copy directly to Vault)
    ├── manifest.json
    ├── main.js
    ├── styles.css
    ├── server.js
    ├── index.html
    ├── ...
```

### Version Number Management

The version number is sourced **exclusively** from the `version` field in `package.json`. When running build scripts, this value is automatically synced to `obsidian/manifest.json`. **Only modify the `package.json` file for version updates.**

## Node Metadata Field Definitions

| Field | Description | Example Value |
|-------|-------------|---------------|
| UUID | Unique identifier | `a1b2c3d4` |
| ID | Node identifier | `n1`, `n1.1` |
| Status | Status | N=new, O=ongoing, D=done, U=update, C=cancel |
| Priority | Priority | 1️⃣~5️⃣, default 2️⃣ |
| Mainline | Mainline flag | true/false |
| Module | Module name | Inherited from parent node |
| AITaskJournal | Task log reference | `[[../prompt/.historylog/xxx.md]]` |

## Multi-Platform Runtime Mode Comparison

| Feature | Standalone Run | VS Code Extension | Obsidian Plugin |
|---------|----------------|-------------------|-----------------|
| Launch Method | `node server.js` / `pnpm start` | Click Activity Bar icon | Click Ribbon Icon |
| Working Directory | Current directory | Configured via folder picker | Vault root + config setting |
| Data Storage | `./todo/` / `./prompt/` | `todo/` / `prompt/` under working directory | `Vault/{dataDir}/todo/` / `prompt/` |
| Frontend Code | Identical | Identical (iframe embedded) | Identical (iframe embedded) |
| Backend Code | server.js | server.js (reused) | server.js (reused) |
| Supported Platforms | All Node.js-supported platforms | Desktop (Mac/Windows/Linux) | Desktop (Mac/Windows), iPad not supported |

## Development Guide

### Tech Stack
- Frontend: Vanilla HTML/CSS/JavaScript (no framework dependencies)
- Markdown Rendering: [marked.js](https://marked.js.org/)
- Table Component: [Tabulator](https://tabulator.info/)
- VS Code API: [Webview API](https://code.visualstudio.com/api/extension-guides/webview)
- Obsidian API: [Plugin API](https://docs.obsidian.md/Reference/TypeScript+API/Plugin)

### Key Files
| File | Description |
|------|-------------|
| `index.html` | Main page containing all core JS logic (~6000+ lines) |
| `server.js` | HTTP server, supports standalone run or being required by VS Code/Obsidian plugins |
| `extension.js` | VS Code extension entry point |
| `obsidian/main.js` | Obsidian plugin entry point |
| `styles/main.css` | Main style sheet |
| `js/config.js` | Reference for global configuration parameters |

### Adding New Features
1. Add functions in the corresponding module section of `index.html`
2. Add UI elements in the relevant HTML section if needed
3. Add style rules in the corresponding section of `styles/main.css` if needed
4. Update the `AGENTS.md` index
5. Re-run `npm run build:release` to generate plugins for both platforms

## Packaging Configuration

`.vscodeignore` is configured to exclude the following files (not packaged into `.vsix`):
- Version control files (`.git/`, `.gitignore`)
- Development documentation (`AGENTS.md`, `DESIGN.md`, `docs/`)
- Package management caches (`node_modules/`, `pnpm-lock.yaml`)
- Runtime data (`todo/`, `data/`, `prompt/.historylog/`)
- Obsidian build source code (`obsidian/`)
- Build scripts (`scripts/`)
- Build outputs (`release/`, `obsidian-dist/`)
